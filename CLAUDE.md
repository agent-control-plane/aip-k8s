# CLAUDE.md — aip-k8s development guide

## Project overview

AIP (Agent Intent Protocol) Kubernetes implementation. A governance control plane
for autonomous agents: gateway (HTTP API), controller (reconciler), dashboard (UI),
and CRDs (AgentRequest, AgentDiagnostic, SafetyPolicy, GovernedResource).

## Build & test commands

```bash
make test                  # unit tests (envtest, no cluster needed)
make test-e2e              # e2e tests (requires Kind cluster: aip-k8s-test-e2e)
make manifests             # regenerate CRDs and RBAC from markers
make generate              # regenerate DeepCopy methods
make lint                  # golangci-lint (config in .golangci.yml)
make fmt                   # go fmt
make build                 # build all binaries
```

Always run `make manifests generate` after changing API types in `api/v1alpha1/`.
Never hand-edit `zz_generated.deepcopy.go` — it is fully generated by `make generate`
and any manual changes will be overwritten or will corrupt the file.

## Architecture

```
cmd/gateway/       — HTTP API server (stateless, talks to K8s API)
cmd/controller/    — controller-runtime reconciler for AgentRequest lifecycle
cmd/dashboard/     — static HTML dashboard (proxies to gateway)
api/v1alpha1/      — CRD type definitions with kubebuilder markers
internal/controller/ — reconciler logic
internal/evaluator/  — CEL policy evaluator
charts/aip-k8s/    — Helm chart
test/e2e/          — Ginkgo e2e test suite
```

## Go conventions (Kubernetes ecosystem)

### File organization
- **Max 500 lines per file.** If a file exceeds this, split by concern. The gateway
  handlers should be split into separate files by resource type (requests, diagnostics,
  governed resources, safety policies).
- **One type per file** for major types. Helper types can share a file with their parent.
- Group imports in three blocks: stdlib, external, internal (enforced by goimports).

### Error handling
- Wrap errors with context: `fmt.Errorf("reconciling AgentRequest %s: %w", name, err)`
- Never ignore errors silently. If intentionally ignoring, add a comment explaining why.
- Use `apierrors.IsNotFound()`, `apierrors.IsConflict()`, `apierrors.IsAlreadyExists()`
  — never check error strings.
- Handle `AlreadyExists` in create-or-update patterns. `retry.RetryOnConflict` only
  retries 409 Conflict, NOT AlreadyExists — these are distinct error classes and must
  be handled separately.

### Logging
- **Controller**: Use `log.FromContext(ctx)` (logr via controller-runtime). Never `log.Printf`.
- **Gateway**: Use `log.FromContext(ctx)` or `slog` with structured fields. Never `log.Printf`.
- Include resource identifiers in every log line: `logger.Info("msg", "name", req.Name, "namespace", req.Namespace)`
- Use `logger.V(1)` for debug-level, `logger.V(2)` for trace-level.
- Never log secrets, tokens, or credentials.

### Controller patterns
- **Patch, don't Update.** Use `client.MergeFrom(obj.DeepCopy())` for status patches.
  Compute the patch base immediately before patching — never reuse a stale base across
  multiple patch calls.
- **APIReader for consistency.** Use `r.APIReader.Get()` when you need a fresh read
  (e.g., before status transitions) to avoid stale informer cache.
- **Idempotent reconciliation.** Every reconcile must be safe to re-run. Check current
  state before acting. Treat NotFound on cleanup as success.
- **Return after status patch.** After calling `Status().Patch()`, return immediately
  (the requeue will get a fresh object). Never continue with a stale object.
- **Injectable dependencies.** Use `Clock func() time.Time` and similar fields for
  testability. Never call `time.Now()` directly in reconcile logic.

### HTTP handler patterns (gateway)
- Apply `http.MaxBytesReader(w, r.Body, 1<<20)` on ALL handlers that read a body.
  No exceptions — including admin endpoints.
- Validate required fields before any K8s API call.
- Return proper HTTP status codes: 400 (bad request), 401 (no auth), 403 (wrong role),
  404 (not found), 409 (conflict/duplicate), 500 (internal error).
- Never block a handler goroutine for more than 30 seconds. Long operations should
  return 202 Accepted with a status polling endpoint.

### Kubernetes API conventions
- Use constants for phase strings, denial codes, condition types. No magic strings.
- CRD field names: camelCase in JSON, PascalCase in Go.
- Use kubebuilder markers for validation (`+kubebuilder:validation:MinLength=1`).
- Use `+kubebuilder:validation:XValidation` for immutability (`self == oldSelf`).
- Printer columns for `kubectl get`: include Phase, Age, and the most useful status field.

### Testing
- **Unit tests**: Use table-driven tests. Mock K8s client with `fake.NewClientBuilder()`
  for simple cases, envtest for reconciler tests.
- **Test file naming**: `foo_test.go` in the same package for white-box tests.
- **Assertions**: Use `require` (fail fast) for setup, `assert` for checks in
  table-driven tests. Or use `if got != want { t.Fatalf(...) }` — both are acceptable.
- **No shared mutable state across tests.** Each test creates its own resources with
  unique names (use `fmt.Sprintf("test-%s-%d", name, time.Now().UnixNano())`).
- **Test edge cases**: AlreadyExists, Conflict, NotFound, empty lists, concurrent access.

### e2e test cleanup
- **AfterAll/cleanup must delete by label selector or `--all`, never by individual name.**
  Deleting resources by name only cleans up what you explicitly created in that phase —
  any resource created with a different name (e.g., from a parallel phase or a prior run)
  will leak and silently break unrelated tests.
  ```bash
  # Wrong — misses resources created with a different name:
  kubectl delete safetypolicy gw-require-human -n $NS --ignore-not-found
  # Correct:
  kubectl delete safetypolicy --all -n $NS --ignore-not-found
  ```
- Each test phase is responsible for cleaning up everything it creates. Assume no ordering
  guarantee between phase teardown and the next phase's setup.

### Helm chart standards
- Never default image tag to `latest`. Use `{{ .Values.image.tag | default .Chart.AppVersion }}`.
- Always set resource requests AND limits.
- Always set security context (runAsNonRoot, readOnlyRootFilesystem, drop ALL capabilities).
- Quote user-supplied values in templates: `{{ .Values.foo | quote }}`.
- Guard optional features with `{{ if .Values.feature.enabled }}`.

### Security
- Validate auth on every mutating endpoint. Health/ready probes are the only exceptions.
- Never trust `X-Remote-User` headers without CIDR validation — and prefer JWT over
  proxy headers in any environment with a service mesh.
- Never log or embed tokens, secrets, or credentials in error messages.
- Use `crypto/rand` for random values, never `math/rand`.

## Common mistakes to avoid

1. **Reusing a `client.MergeFrom` patch base** after the object has been patched.
   Always recompute or return-and-requeue.
2. **Forgetting RBAC** when adding new K8s resource access. Update both the
   ClusterRole in the Helm chart AND run `make manifests` for controller RBAC markers.
3. **Missing `AlreadyExists` handling** in create-if-not-exists patterns inside
   `retry.RetryOnConflict` loops.
4. **Hardcoded timeouts and magic numbers.** Use named constants or configurable flags.
5. **`//nolint:dupl`** without justification. If handlers look identical, prefer extracting
   a generic helper. If the duplication is intentional (different behavior despite similar
   structure), document why with a comment.
6. **`goto` for control flow.** Use helper functions that return booleans instead.
7. **Hand-editing generated files.** Never edit `zz_generated.deepcopy.go` directly —
   run `make generate`. Edits will be overwritten and intermediate states can corrupt
   the file silently (e.g., dropping struct fields).
8. **Named-only cleanup in e2e AfterAll.** See e2e test cleanup section above.
